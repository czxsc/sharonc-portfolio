# Personal Portfolio

## Project Overview

This project is a fully custom-coded personal portfolio website.

It originally began as a Framer prototype, but the goal is to recreate and expand it through code rather than relying on a website builder. The objective is not simply to reproduce the Framer version, but to evolve it into something that feels intentionally designed and handcrafted.

The website should communicate two things simultaneously:

- strong technical competence
- thoughtful visual design

The overall impression should be:

> "A software engineer who deeply appreciates design."

This should **not** resemble a generic React portfolio template.

---

# Primary Design Goals

The design should feel:

- minimal
- editorial
- clean
- premium
- intentional
- spacious

Rather than maximizing information density, prioritize composition, whitespace, hierarchy, and typography.

Every section should feel designed rather than assembled.

When making design decisions, favor restraint over decoration.

---

# Overall Style

The aesthetic is heavily inspired by modern designer portfolios.

Key references:

- Japanese / Korean portfolio websites
- editorial magazine layouts
- Framer showcase sites
- print-inspired typography
- asymmetric compositions
- subtle playful illustrations

The personality should be approximately:

85% professional

15% personal

The personality should come from small details—not loud graphics.

Examples include:

- coffee illustrations
- cat doodles
- hand-drawn icons
- tiny hover interactions
- slightly imperfect layouts
- subtle stickers or paper textures

Avoid making the website feel cute.

The goal is sophisticated with hints of personality.

---

# Color Palette

Primary background:

warm off-white

Examples:

- #F8F5F2
- #F6F3EF

Avoid pure white.

Text:

Near black.

Examples:

#1B1B1B

Accent colors should be extremely limited.

Small muted browns, dark greens, or navy accents are acceptable.

Never introduce bright saturated colors unless they are part of project imagery.

---

# Typography

Typography is the primary visual element.

Use strong hierarchy.

Large bold headlines.

Small editorial body text.

Comfortable line lengths.

Generous spacing.

Do not rely on excessive font weights or colors for hierarchy.

Hierarchy should come from:

- size
- spacing
- positioning

Avoid trendy oversized outlined fonts or flashy effects.

---

# Layout Principles

Layouts should rarely be perfectly centered.

Prefer:

- asymmetry
- offset elements
- negative space
- editorial grids

Sections should breathe.

Do not stack content like a Bootstrap landing page.

Each section should feel composed independently.

Use whitespace intentionally.

---

# Motion

Animations should be subtle.

Examples:

- gentle fades
- slight movement
- smooth image reveals
- elegant hover states

Avoid:

- bouncing
- elastic motion
- flashy transitions
- excessive parallax

Everything should feel calm.

---

# Component Philosophy

Favor reusable components without making everything look identical.

Cards should have variation.

Image layouts should vary naturally.

Sections should not become repetitive.

Avoid obvious "template" patterns.

---

# Website Structure

Current planned sections:

1. Hero

Large name.

Short introduction.

Portrait.

Small decorative illustrations.

Primary navigation.

---

2. About

Short introduction.

Current focus.

Design philosophy.

Not a long biography.

---

3. Featured Projects

Primary focus of the website.

Projects should receive significantly more visual attention than other sections.

Each project should include:

- title
- short description
- technologies
- role
- links
- large imagery

Project layouts may alternate to avoid repetition.

---

4. Experience

Minimal timeline.

Readable.

Simple.

Avoid resume-like walls of text.

---

5. Interests

A lightweight section showing personality.

Possible items:

- coffee
- cats
- piano
- photography
- travel
- sketching
- favorite objects

This section should feel playful without distracting from the professional tone.

---

6. Contact

Simple.

Minimal.

Easy to find.

---

# Design Details

Small delightful details are encouraged.

Examples:

tiny doodles

paper stickers

coffee stains

handwritten arrows

cat illustrations

tape graphics

light textures

These should be sparse.

Less is more.

---

# Project Presentation

Projects are the centerpiece.

Each project should feel like a mini case study rather than a gallery item.

Large imagery is preferred.

Avoid showing every project equally.

Prioritize quality over quantity.

---

# Responsiveness

Desktop is the primary experience.

Tablet should remain elegant.

Mobile should preserve the visual identity rather than simply stacking everything vertically.

Layouts may be reorganized rather than directly scaled down.

---

# Accessibility

Maintain:

- semantic HTML
- keyboard navigation
- sufficient color contrast
- reduced-motion support

Accessibility should never compromise the overall aesthetic.

---

# Technical Philosophy

Prefer simplicity over unnecessary abstraction.

Build reusable components where appropriate.

Avoid overengineering.

Keep animations performant.

Favor CSS over animation libraries unless a library clearly improves maintainability.

---

# Things to Avoid

Do NOT make this resemble:

- default React portfolios
- Tailwind UI templates
- developer landing pages
- startup SaaS websites
- Material UI dashboards

Avoid:

- giant gradient backgrounds
- glowing buttons
- glassmorphism
- neumorphism
- excessive shadows
- overusing rounded corners
- large colorful icons
- generic hero illustrations

The website should feel timeless rather than trendy.

---

# Guiding Principle


Every design decision should support this question:

"Would this look believable if it belonged to an experienced product designer who also happens to be a software engineer?"

If not, simplify.
